What is Dcanary?

Dcanary combines the power of decentralized infrastructure with modern CI/CD practices

The Name Behind the System

Daniel (D)

Named after its creator, Daniel - representing personal innovation and dedication to decentralized technologies.

Canary

Inspired by the "canary in the coal mine" concept - early detection of issues in your build pipeline before they become critical problems.

Core Philosophy

Trustless: No single point of failure or control
Tamper-proof: Immutable build verification on blockchain
Decentralized: Powered by Internet Computer Protocol
Developer-friendly: Easy setup and intuitive commands

Powerful Features

Everything you need for secure, scalable, and reliable CI/CD operations

Multi-Executor Verification

Run builds on multiple independent executors and compare results to detect tampering or environment issues.

SCM Integration

Seamless integration with GitHub, GitLab, and other SCM providers with automatic webhook triggers.

Powerful CLI

Complete command-line interface called "Mody" for all pipeline management operations.

Cryptographic Security

All build results are cryptographically hashed and verified for maximum security and integrity.

Immutable Audit Trail

Complete history of all builds and verifications stored immutably on the Internet Computer.

Internet Computer Powered

Built on ICP for true decentralization, scalability, and resistance to censorship.

Quick Installation

Get up and running with Dcanary in minutes

Prerequisites

Node.js 18+

Latest LTS version recommended for optimal performance

DFX (Dfinity SDK)

Essential for Internet Computer development and deployment

Git

Required for version control and repository operations

Installation Steps

1 Clone the repository
git clone https://github.com/username/dcanary.git
2 Navigate to directory
cd dcanary/Dcanary
3 Install dependencies
npm install
4 Start local replica
dfx start --background
5 Deploy canisters
dfx deploy
6 Install Dcanary CLI
cd cli && npm install -g .

Quick Start Guide

Get up and running with Dcanary in minutes. For comprehensive documentation, visit our detailed docs.

Essential Commands

The core commands you need to get started with Dcanary

Add Build Instructions

mody add-instructions --project-id "my-project" --version "1.0.0" --file "./build.sh"

Register your project's build instructions with the system

Request Verification

mody request-verification --project-id "my-project" --version "1.0.0"

Request build verification across multiple executors

Setup SCM Integration

mody scm register "my-project" --provider github --owner "username" --repo "repo-name"

Connect your GitHub repository for automatic builds

Get Status

mody get-status --project-id "my-project"

Check project status and build history

Configure CLI

mody configure --network "local" --identity "default"

Configure CLI settings and network connection

Version Info

mody version

Display CLI version and system information

Real-World Examples

See how Dcanary works with different project types and workflows

TypeScript Web Application

1
Setup Project

mody add-instructions --project-id "my-webapp" --version "1.0.0" --file "./build.sh"

2
Register GitHub Repository

mody scm register "my-webapp" --provider github --owner "myuser" --repo "webapp"

3
Request Verification

mody request-verification --project-id "my-webapp" --version "1.0.0" --executors 3

Node.js API Server

1
Create Build Script

#!/bin/bash
npm ci
npm run test
npm run build
docker build -t api-server .

2
Add Instructions

mody add-instructions --project-id "api-server" --version "2.1.0" --file "./api-build.sh"

3
Setup Webhook

mody webhook configure --url "https://canister.ic0.app/webhook" --secret "secure-secret"
